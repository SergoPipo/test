# UX-макет: Аналитика бэктеста — гистограмма P&L + donut Win/Loss (для задачи 7.16)

> **Статус:** плейсхолдер. Заполняется UX-агентом на W0 по `prompt_UX.md` задача 4.
> Источники макета:
> - `Sprint_7/sprint_design.md` Приложение D
> - `Sprint_7/prompt_UX.md` секция «Задачи W0 / 4»
> - Память `project_s7_interactive_zones.md` — спецификация заказчика дословно

## Содержание (заполнить UX-агенту)

1. **Описание** — два графика рядом на вкладке «Обзор» страницы результатов бэктеста (стиль TradingView).

2. **Гистограмма «Распределение прибылей и убытков»**
   - Бакеты по % P&L (например: -2%, -1.5%, -1%, -0.5%, 0%, +0.5%, +1%, +1.5%, +2%, +2.5%, +3%, +3.5%, +4%, +4.5%).
   - Высота столбика — кол-во сделок в бакете.
   - Цвет: красные = убытки, зелёные = прибыли.
   - Пунктирные вертикальные линии: средний убыток (красная пунктирная) и средняя прибыль (зелёная пунктирная) с подписями.
   - Tooltip при hover: «Бакет +1%–+1.5%: 8 сделок».

3. **Donut-чарт «Соотношение побед/поражений»**
   - 3 сегмента: Победы (зелёный), Убытки (красный), Безубыточность (жёлтый).
   - В центре donut: общее число сделок крупно.
   - Справа от donut: легенда с кол-вом и % для каждого сегмента.

4. **Расположение** — два графика рядом (50%/50% ширины) на вкладке «Обзор».

5. **Источник данных** — `trades[]` из `BacktestResult`. Используется существующая модель.

6. **Интерактивные зоны на графике (отдельный аспект 7.16)**
   - Hover на зону входа/выхода сделки → зона подсвечивается белым контуром, цвет ярче.
   - Клик → открывается панель деталей сделки под графиком (entry_reason, exit_reason, P&L).

7. **States**
   - Гистограмма/donut при `trades.length === 0` — empty state «Нет сделок для анализа».
   - Loading skeleton при загрузке `BacktestResult`.

8. **ASCII-mockup** обоих графиков рядом + панели деталей сделки.

9. **Контракт с backend** — нет дополнительных endpoints (используется `BacktestResult.trades`). Возможно потребуется добавить `entry_reason` / `exit_reason` в `TradeRecord` (зависимость от backend, согласовать с ARCH).

10. **Тестовые сценарии для QA** — рендер гистограммы; рендер donut; hover/click на зоне; переключение между бэктестами без утечки графиков.
