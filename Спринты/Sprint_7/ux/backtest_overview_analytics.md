> **Статус:** заполнен UX-агентом на W0 (2026-04-25).
> **Источники:** `Sprint_7/sprint_design.md` Приложение D, `Sprint_7/prompt_UX.md` задача 4, память `project_s7_interactive_zones.md` (спецификация заказчика дословно).

## 1. Описание сценария

Пользователь открывает страницу результата бэктеста `/backtest/{id}` (или `/backtest/{id}/overview`). На вкладке «Обзор» — два графика рядом, дающие быстрый визуальный анализ распределения сделок и win-rate стратегии. Дополнительно — на основном ценовом графике зоны входа/выхода интерактивны (hover/click → деталь сделки). Фактические данные — из `BacktestResult.trades[]` (модель уже существует с S5).

Задача памяти: `project_s7_interactive_zones.md` определяет визуал гистограммы и donut, а также интерактивные зоны.

## 4. Гистограмма «Распределение P&L»

### Данные

- Bucketize `trades[].pnl_pct` (или `pnl` если pct не доступен) с шагом 0.5%.
- Диапазон: от `min(pnl_pct)` округлённого вниз до `max(pnl_pct)` вверх. Минимум 6 buckets (если разброс мал — расширить).
- Y — кол-во сделок в bucket; X — bucket label («-1.5%..-1%»).

### Визуал

- Mantine `<BarChart>` (или Recharts/Visx) с custom colors:
  - Bucket с центром `< 0` — `var(--mantine-color-red-6)`.
  - Bucket с центром `>= 0` — `var(--mantine-color-green-6)`.
  - Безубыточные (например, `[-0.05%, +0.05%]`) — `var(--mantine-color-yellow-5)`.
- Пунктирные вертикальные линии:
  - **Avg loss** = среднее по убыточным сделкам — красная пунктирная (`stroke-dasharray: 4 4`), подпись «Ср. убыток: -1.2%».
  - **Avg profit** = среднее по прибыльным — зелёная пунктирная, подпись «Ср. прибыль: +1.8%».
- Tooltip при hover на бар: `Бакет +1%–+1.5%: 8 сделок, Σ PnL: +9 200 ₽`.

### States

- `loading` — `<Skeleton h={240} />`.
- `empty` (`trades.length === 0`) — placeholder «Нет сделок для анализа» + иконка.
- `error` — «Не удалось загрузить данные сделок» + retry.

### Доступность

- Каждый бар — `<rect role="img" aria-label="Бакет +1%—+1.5%, 8 сделок">`.
- Color-blind safe — паттерны (диагональные штрихи для loss, точки для profit) при `prefers-contrast: more`.

## 5. Donut «Win/Loss/BE»

### Данные

- `wins = trades.filter(t => t.pnl > epsilon).length`.
- `losses = trades.filter(t => t.pnl < -epsilon).length`.
- `be = trades.length - wins - losses`. (epsilon: `0.01%` или `1₽`, подбирается, по умолчанию pnl_pct ≈ 0.05%).

### Визуал

- Mantine `<DonutChart>` или recharts `<PieChart>`. Радиусы: outer 96, inner 56.
- 3 сегмента с цветами green-6 / red-6 / yellow-5.
- В центре: `<Text size="xl" fw={700}>{trades.length}</Text>` + subline «сделок».
- Справа — легенда (Mantine `<DonutChart.Legend>`):
  - `■ Победы — 72 (58%)`
  - `■ Убытки — 46 (37%)`
  - `■ Безубыток — 6 (5%)`

### States

- `loading` — skeleton circle.
- `empty` — donut не рендерится, текст «Нет сделок».
- Если `losses === 0 && wins === 0` (только BE) — donut всё равно рендерим (yellow только).

## 6. Интерактивные зоны на ценовом графике

Раздел из памяти `project_s7_interactive_zones.md`:

- На ценовом графике (вкладка «График» или мини в «Обзоре») трейды отрисованы как:
  - Маркер входа (зелёный треугольник вверх для long entry / красный — для short).
  - Маркер выхода (X / квадрат, цвет зависит от pnl).
  - Полупрозрачная зона между entry и exit (color по знаку pnl).
- **Hover на зоне** → зона подсвечивается белым контуром (1px), цвет ярче (alpha 0.35 → 0.55).
- **Click на зоне** → внизу графика разворачивается панель `TradeDetailsPanel`:
  - `Entry: 2026-04-10 14:00, цена 305.5 ₽, причина: «RSI < 30 + bullish engulfing»`.
  - `Exit: 2026-04-12 11:00, цена 312.0 ₽, причина: «Take Profit»`.
  - `P&L: +6.5 ₽ (+2.13%)`.
  - `Длительность: 1д 21ч`.
  - Ссылка «Открыть свечу в графике» (центрирует timeScale на entry).
- Esc / повторный клик закрывает панель.

`assume(ARCH)`: `entry_reason` и `exit_reason` в `TradeRecord` могут отсутствовать в S5 модели — нужна миграция. ARCH должен в `arch_design_s7.md` подтвердить либо: (а) добавить поля через Alembic, (б) использовать существующий `signal.metadata` JSON. UX готов адаптировать.

## 8. Поведение

| Событие | Реакция |
|---------|---------|
| Mount страницы | Fetch `BacktestResult` (REST), вычисление buckets/agg на frontend. |
| Hover на бар гистограммы | Tooltip с детализацией. |
| Hover на сегмент donut | Highlight + tooltip. |
| Hover на зоне трейда (chart) | Подсветка. |
| Click на зоне | Открытие `TradeDetailsPanel`. |
| Click «Открыть в графике» | timeScale `setVisibleRange(entry-N, exit+N)`. |
| Сменa бэктеста (другая ссылка) | Cleanup graph (см. lightweight-charts gotcha) + новый fetch. |

## 9. Связь с другими экранами

- `/backtest/{id}/trades` — таб со всеми сделками; клик на bar/segment → переход с фильтром (опционально).

## 10. Контракт с backend

- `GET /api/v1/backtest/{id}` → существующая модель `BacktestResult.trades[]`. Дополнительные поля для интерактивных зон:
  - `entry_reason: string` (новое поле — `assume(ARCH)`, миграция).
  - `exit_reason: string` (новое поле — `assume(ARCH)`).
  - `entry_ts, exit_ts, entry_price, exit_price, pnl, pnl_pct, duration_seconds`.
- Гистограмма и donut — **frontend-only вычисления**, не требуют backend агрегатов.

Если ARCH решит, что reason не вносим в S7 (out of scope) — UX оставляет панель деталей без поля «Причина», текст «—». Не блокирует.

## 11. Доступность

- DonutChart с `aria-label` для каждого сегмента.
- BarChart — каждый bar `<rect>` с `aria-label`.
- TradeDetailsPanel — `role="dialog"`, focusable, `Esc` закрывает.
- Цветовая схема — color-blind safe (дублирование смыслов через паттерны/иконки ↑↓).
- Keyboard: Tab по барам гистограммы (опционально), Enter — selection.

## 12. Тестовые сценарии для QA

(в `e2e_test_plan_s7.md`, секция 7.16)

1. Открыть бэктест с ≥10 сделками → видны гистограмма и donut.
2. Гистограмма: красные бары для убытков, зелёные для прибылей, пунктирные средние линии видны.
3. Donut: 3 сегмента (или меньше при отсутствии BE), сумма % = 100±1%.
4. Hover на бар → tooltip с детализацией.
5. Hover на зоне трейда (chart) → подсветка.
6. Click на зоне → `TradeDetailsPanel` открыт с корректными `entry/exit reason`, `pnl`.
7. Esc → панель закрыта.
8. Бэктест без сделок → empty state на обоих графиках.
9. Переключение между двумя бэктестами — графики корректно очищаются (без утечки lightweight-charts).
10. Mobile <1024 — графики стекуются (1 колонка).

- `backtest-trade-detail-entry-reason`, `backtest-trade-detail-exit-reason` — оставлены как есть (детали панели).

## 13. Stack Gotchas

- **lightweight-charts series cleanup** — при смене бэктеста / unmount.
- **bucket-edge inclusivity** (новый кандидат на gotcha): чётко определить — bucket `[-0.5%, 0%)` или `(-0.5%, 0%]`. Иначе нулевые/граничные сделки попадают то туда, то сюда. UX рекомендует `[low, high)` (включая нижнюю границу).
- **Decimal precision** — backend шлёт Decimal как строку (известная gotcha S5R); парсить через `toNum()` перед расчётом buckets.
