## Содержание (заполнить UX-агенту)

1. **Описание** — при вводе `/` в input AI-чата открывается dropdown с 5 командами; выбранная команда + аргумент превращаются в «контекстный chip» в сообщении пользователя; backend подгружает данные по контексту в enriched prompt.

2. **5 команд**
   - `/chart TICKER [TF]` — добавить контекст графика (например, `/chart EURUSD 1H`).
   - `/backtest BACKTEST_ID` — метрики бэктеста.
   - `/strategy STRATEGY_ID` — код стратегии.
   - `/session SESSION_ID` — состояние торговой сессии.
   - `/portfolio` — текущий портфель (без аргументов).

3. **Dropdown UI**
   - Открывается при вводе `/` в начале строки или после пробела.
   - Список команд с подсветкой при keyboard navigation (↑/↓).
   - Каждая строка: имя команды + краткое описание + пример аргумента.
   - Filter — при наборе `/cha` показываются только подходящие.
   - Enter / клик → выбор команды; курсор после имени команды + пробел.

4. **Аргумент**
   - Подсказка справа от input: «Введите тикер» (для /chart) или «Введите ID бэктеста».
   - Backend валидация — frontend отправляет, backend проверяет ownership и возвращает 404/403 при невалидном.

5. **Контекстный chip в сообщении**
   - При нажатии Enter / Send команда + аргумент превращается в chip с иконкой типа:
     - 📊 Chart: EURUSD, 1H
     - 📈 Backtest: #42
     - 📜 Strategy: #15
     - 💹 Session: #7
     - 💼 Portfolio
   - Chip кликабельный — переход на ресурс в новой вкладке (`/chart?ticker=EURUSD&tf=1H` и т.д.).
   - Текст пользователя (после команды) — обычным текстом рядом с chip.

6. **States**
   - Dropdown открыт / скрыт.
   - Команда выбрана, ждёт аргумент.
   - Chip отображается в input, пользователь дописывает текст.
   - Ошибка валидации (id не принадлежит юзеру) — красный chip + tooltip «Нет доступа».

7. **Множественный контекст**
   - Можно добавить несколько команд в одном сообщении: `/chart EURUSD /backtest 42` → 2 chip'а.
   - В `ChatRequest.context` отправляется массив элементов.

8. **ASCII-mockup** input + dropdown + сообщение с chip'ами.

9. **Контракт с backend** — `POST /api/v1/ai/chat` с расширенным `context: [{type, id}]` (контракт C3). Backend подгружает данные и формирует enriched prompt.

10. **Тестовые сценарии для QA**
    - Ввёл `/` → dropdown показывается.
    - Выбрал `/chart EURUSD` → enriched prompt отправлен → AI ответил с контекстом тикера.
    - `/strategy 999` где id принадлежит другому юзеру → 403, красный chip.
    - Множественный контекст в одном сообщении.
