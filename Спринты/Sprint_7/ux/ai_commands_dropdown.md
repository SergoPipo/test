# UX-макет: AI слэш-команды dropdown (для задач 7.18 / 7.19)

> **Статус:** заполнен UX-агентом на W0 (2026-04-25).
> **Источники:** `Sprint_7/sprint_design.md` Приложение D, `Sprint_7/prompt_UX.md` задача 6, `.claude/plans/purring-herding-badger.md` (план AI-команд), память `project_slash_commands.md`, контракт C3 (Приложение A).
> **Связь с задачей S7:** 7.18 (backend — расширение `ChatRequest.context[]` / enriched prompt) + 7.19 (frontend — dropdown + chips).

---

## 1. Описание сценария

В `/ai/chat` пользователь ведёт диалог с AI. При вводе `/` (в начале строки или после пробела) появляется dropdown с 5 командами. Пользователь выбирает команду + указывает аргумент (тикер / id) → команда превращается в **контекстный chip** в input. На Enter сообщение уходит на backend через `POST /api/v1/ai/chat` с расширенным `context: [{type, id}]`. Backend подгружает данные по контексту (свежие свечи, метрики бэктеста, код стратегии и т.д.) и формирует enriched prompt для модели.

Цель — дать AI прямой доступ к артефактам пользователя (графикам, бэктестам, стратегиям) без копирования вручную.

## 2. Список 5 команд

| Команда | Аргумент(ы) | Описание | Иконка |
|---------|-------------|----------|--------|
| `/chart` | `TICKER [TF]` | Контекст графика (последние свечи + текущая цена) | 📊 (chart-bar) |
| `/backtest` | `BACKTEST_ID` | Метрики и результаты бэктеста | 📈 (trending-up) |
| `/strategy` | `STRATEGY_ID` | Код / blocks XML стратегии | 📜 (file-code) |
| `/session` | `SESSION_ID` | Состояние торговой сессии (PnL, позиции, события) | 💹 (activity) |
| `/portfolio` | — (без аргументов) | Текущий портфель пользователя | 💼 (briefcase) |

(Иконки эмодзи здесь — для текстового макета. В коде используются Tabler-icons, без эмодзи в коде.)

## 3. Dropdown UI

```
┌─ Сообщение AI ──────────────────────────────────────────┐
│  Введите запрос или команду /                            │
│  ┌─ Popover ─────────────────────────────────────────┐  │
│  │ /chart       Контекст графика                      │  │
│  │              пример: /chart SBER 1H                │  │
│  │ /backtest    Метрики бэктеста                      │  │
│  │              пример: /backtest 42                  │  │
│  │ /strategy    Код стратегии                         │  │
│  │              пример: /strategy 15                  │  │
│  │ /session     Состояние сессии                      │  │
│  │              пример: /session 7                    │  │
│  │ /portfolio   Текущий портфель                      │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│                                            [Отправить ▶]│
└──────────────────────────────────────────────────────────┘
```

- Popover — Mantine `<Combobox>` или `<Menu>`, открывается при `/` в начале строки или после пробела.
- Filter — при наборе `/cha` — остаётся только `/chart`. Использовать `useCombobox`.
- Keyboard navigation: `↑/↓` — выбор; `Enter` / `Tab` — подставить команду + пробел; `Esc` — закрыть.
- Mouse: click на строку — то же.

## 4. Аргумент

После выбора команды (например, `/chart`) input выглядит:

```
/chart |
        ↑ cursor
```

Подсказка справа от input (Mantine `<Input.Description>` или inline ghost):

```
/chart [TICKER] [TF?]   ← inline placeholder
```

- Для `/chart` — нужно минимум TICKER. TF опциональна (default — текущий tf графика, либо 1D).
- Для `/backtest`, `/strategy`, `/session` — нужен ID (число).
- Для `/portfolio` — никаких аргументов.

Опциональный inner-dropdown «picker»:
- При вводе `/chart ` — показать popover с поиском тикеров (использует существующий ticker-search компонент).
- При вводе `/backtest ` — показать список последних 10 бэктестов user'а с краткой инфой.
- При вводе `/strategy ` — список стратегий.
- При вводе `/session ` — список активных + последних завершённых сессий.

(MVP в S7 — без inner-picker, ввод текстом. Picker — out of scope для S7, в `Sprint_7/sprint_state.md` отметить как nice-to-have.)

## 5. Контекстный chip в сообщении

Когда пользователь нажимает Enter (или Send) с командой в input:

```
┌─ Сообщение пользователя ────────────────────────────────────┐
│ [📊 Chart: SBER, 1H]  объясни что произошло на этом графике │
└──────────────────────────────────────────────────────────────┘
```

- Команда + аргумент превращаются в `<Badge>` (Mantine `<Badge variant="light" color="blue" leftSection={<Icon/>}>`).
- Текст после команды — обычным текстом.
- Chip кликабельный → переход к ресурсу:
  - `/chart?ticker=SBER&tf=1H`
  - `/backtest/42`
  - `/strategy/15`
  - `/sessions/7`
  - `/portfolio` (или `/account`)
- Chip в **отображении сообщения в истории чата** — тот же визуал, ссылка работает.

Множественный контекст:
```
Сообщение: /chart SBER /backtest 42  сравни цену сейчас с тем что было в бэктесте
Render:   [📊 Chart: SBER]  [📈 Backtest: #42]  сравни цену сейчас с тем что было в бэктесте
```

## 6. States

| State | Поведение |
|-------|----------|
| Idle (нет `/`) | Dropdown скрыт |
| Введён `/` в start или after space | Dropdown открыт со всеми 5 |
| Введён `/cha` | Dropdown отфильтрован (показан `/chart`) |
| Команда выбрана, аргумент ожидается | Dropdown закрыт, в input текст команды + cursor + ghost placeholder |
| Аргумент введён (валидный) | Chip preview в input на месте `/cmd arg` (опц., можно делать только при отправке) |
| Аргумент не валиден (например, ID не number) | Inline error «Ожидался ID (число)» |
| Backend 403/404 (ownership) | Chip в чате — красный с tooltip «Нет доступа к ресурсу» |
| Сообщение отправлено | Чип отображается в истории сообщений |

## 9. Контракт с backend (C3)

**Поставщик:** BACK2. **Потребитель:** FRONT1.

- `POST /api/v1/ai/chat` body:
  ```json
  {
    "message": "сравни цену сейчас с тем что было в бэктесте",
    "context": [
      {"type": "chart", "id": "SBER", "tf": "1H"},
      {"type": "backtest", "id": 42}
    ],
    "thread_id": "uuid?"
  }
  ```
- Backend:
  - Проверяет ownership ресурсов (403 если чужой strategy/backtest/session).
  - Подгружает данные (свечи / метрики / код / позиции / portfolio).
  - Формирует enriched prompt и отправляет в LLM.
  - Возвращает ответ.
- Response:
  ```json
  {
    "thread_id": "uuid",
    "messages": [{ "role": "assistant", "content": "...", "metadata": {} }]
  }
  ```

Frontend dispatcher:
- При парсинге message — выделяет `/cmd ARG` сегменты, превращает в `context[]` и оставшийся text — в `message`.
- В UI хранит исходный raw-текст для удобства редактирования.

## 10. Доступность

- Combobox — встроенная a11y Mantine (`role="combobox"`, `aria-expanded`).
- Каждый item в dropdown — `role="option"` с `aria-label`.
- Chip в сообщении — `<a role="link">` с `aria-label="Контекст: график SBER 1H, перейти к графику"`.
- Hotkey `/` для активации dropdown — фокус остаётся на input.
- Skip dropdown — `Esc`.

## 12. Stack Gotchas

- **Combobox + IME composition** (русский ввод): следить за `compositionstart/end`, чтобы `/` не активировал dropdown во время сборки иероглифа/диакритики (см. Mantine known behavior).
- **XSS в chip** (если backend вернёт `entity_label` с user-content): рендерить через safe React, не `dangerouslySetInnerHTML`.
- **403/404 vs ownership leak**: backend должен возвращать 403 для чужих ресурсов и 404 для несуществующих, **не путая два кейса** (ownership leak — кандидат на gotcha).
