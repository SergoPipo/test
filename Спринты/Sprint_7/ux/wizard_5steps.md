# UX-макет: First-run wizard 5 шагов (для задачи 7.8)

> **Статус:** плейсхолдер. Заполняется UX-агентом на W0 по `prompt_UX.md` задача 2.
> Источники макета:
> - `Sprint_7/sprint_design.md` Приложение D
> - `Sprint_7/prompt_UX.md` секция «Задачи W0 / 2»

## Содержание (заполнить UX-агенту)

1. **Описание** — модальное окно с 5 шагами, нельзя пропустить, показывается на первом логине.
2. **Шаг 1: Приветствие** — текст, лого, кнопка «Дальше».
3. **Шаг 2: Дисклеймер о рисках** — длинный текст, обязательная галка «Я прочитал и понимаю», кнопка «Дальше» disabled пока галка снята.
4. **Шаг 3: Выбор брокера / режим** — radio paper/real, dropdown брокера.
5. **Шаг 4: Настройка уведомлений** — Telegram (bot_token), Email, In-app (всегда вкл).
6. **Шаг 5: Финиш** — «Готово, начать работу».
7. **Прогресс-бар** — Mantine Stepper или custom.
8. **Кнопки навигации** — Назад / Дальше / Завершить.
9. **States каждого шага** — пустой, заполненный, ошибка валидации.
10. **ASCII-mockup** для каждого из 5 шагов.
11. **Контракт с backend** — `POST /api/v1/users/me/wizard/complete`, `GET /api/v1/users/me` отдаёт `wizard_completed_at`. Контракт C6.
12. **Тестовые сценарии для QA** — новый юзер видит wizard, после complete не показывается, нельзя пропустить дисклеймер.
