> **Источники:** `Sprint_7/sprint_design.md` Приложение D, `Sprint_7/prompt_UX.md` задача 1, `Спринты/ui_checklist_s5r.md`.
> **Связь с задачей S7:** 7.7 «Виджеты главной страницы (баланс, health, активные позиции)».
> **Ожидание ARCH:** требует подтверждения после публикации `arch_design_s7.md` — backend-эндпоинты для агрегатов (баланс/sparkline, health, активные позиции). См. секцию «Контракты с backend» ниже, помечено `assume(ARCH)`.

## 1. Описание сценария

Пользователь после логина (либо при клике на «Главная» в Sidebar) попадает на страницу `/dashboard`. Это «командный мостик» терминала: сразу видна сводная картина за день — баланс, состояние систем (CB / T-Invest / Scheduler), активные торговые позиции. Виджеты — стартовая навигация: клик по виджету ведёт на детальный экран (`/account`, `/trading`, `/sessions`).

Замена/расширение текущего стартового экрана (в S5R/S6 главная страница — список стратегий/сессий без агрегата). Дашборд показывается всем пользователям, прошедшим wizard (`wizard_completed_at` is not null) — при незавершённом wizard срабатывает редирект на 7.8.

## 3. Виджет «Баланс»

```
┌─ Баланс ─────────────────────────────── Card ──┐
│ ₽ 1 234 567,89                                  │  ← Title 32px
│ ▲ +12 340,11   (+1,01%)   за день  [↑ green]   │  ← Subtitle, диф %
│                                                  │
│ ╭────────────────────────────╮                  │
│ │ ▁▂▂▃▄▄▅▆▆▇█ sparkline      │                  │
│ ╰────────────────────────────╯                  │
│                                                  │
│ Открыть счёт →                                  │
└──────────────────────────────────────────────────┘
```

- **Значение** — общий портфельный баланс (RUB по умолчанию + тогглер валют RUB/USD/EUR; мульти-валютность не складывается в одну сумму, см. ui_checklist S5R).
- **Sparkline** — 24h timeline (lightweight-charts area series или Mantine `<Sparkline>`), цвет совпадает с трендом (green/red).
- **Период** — фиксированный «за день»; на hover sparkline tooltip показывает точку (время, значение).
- **Клик на карточку** — переход в `/account` (вкладка «Позиции»).

States:
- `loading` — `<Skeleton h={32} mb="sm" />` для значения, `<Skeleton h={56} />` для sparkline.
- `empty` — «Нет данных по балансу. Подключите счёт.» + кнопка «Подключить» → `/settings/broker`.
- `error` — «Не удалось получить баланс» + кнопка «Повторить» (retry).
- `multi-currency` — переключатель валют (Mantine `<SegmentedControl size="xs">`) над значением.

## 4. Виджет «Health»

```
┌─ Состояние систем ─────────────────── Card ──┐
│  ●  Circuit Breaker   GREEN   (норма)         │  ← светофор
│  ●  T-Invest          ONLINE  (last 2s)       │
│  ●  Scheduler         RUNNING (12 jobs)       │
│                                                │
│  Подробнее в логах →                           │
└────────────────────────────────────────────────┘
```

- Каждая строка — `Group gap="sm"` с цветной точкой (`Indicator` Mantine, диаметр 12px).
- Цвета:
  - **GREEN** — `var(--mantine-color-green-6)` (норма, online, running).
  - **YELLOW** — `var(--mantine-color-yellow-5)` (warning, частично работает; CB warn-level; reconnect).
  - **RED** — `var(--mantine-color-red-6)` (triggered, offline, stopped).
- На hover точки — tooltip с расшифровкой («Последний цикл: 2 сек. назад», «CB triggered: max_loss=-3%»).
- Клик «Подробнее в логах» → `/logs` с фильтром по типу.
- Источник:
  - CB-state — `app/circuit_breaker/` (по сессиям пользователя — агрегат).
  - T-Invest connection — `app/broker/tinvest/multiplexer.py` heartbeat.
  - Scheduler — `app/scheduler/` (количество active jobs).

States:
- `loading` — три серые точки + skeleton.
- `error` — «Состояние систем недоступно» (карточка остаётся, но точки серые).

## 5. Виджет «Активные позиции»

```
┌─ Активные позиции (3) ─────────────── Card ──┐
│  SBER  ▁▂▃▂▃▃▂  +3 240 ₽  (+1,2%)   [green] │
│  GAZP  ▆▅▅▄▄▄▃   -890 ₽  (-0,4%)    [red]   │
│  LKOH  ▂▃▃▄▄▅▅  +12 100 ₽ (+0,6%)   [green] │
│                                                │
│  Все позиции →                                 │
└────────────────────────────────────────────────┘
```

- Список до 5 позиций, сортировка по abs(P&L) убыв.
- Каждая строка — `<Group justify="space-between">`: `<TickerLogo>` + ticker, sparkline (24h цены), P&L абсолют + %, цвет по знаку.
- Клик на строку → `/chart?ticker=SBER` (открыть график).
- Клик «Все позиции» → `/account` таб «Позиции».

States:
- `loading` — 3 строки skeleton.
- `empty` — «Нет активных позиций. Запустите торговую сессию.» + кнопка «К сессиям» → `/trading`.
- `error` — «Не удалось получить позиции» + retry.
- `>5 позиций` — отображаем 5 + строка «и ещё N…» как ссылка.

## 7. Поведение

| Событие | Реакция |
|---------|---------|
| Hover на карточке | `box-shadow` повышается (Mantine `shadow="md"` → `"lg"`), курсор pointer. |
| Click на карточке | Переход на детальный экран. |
| Hover на sparkline | Tooltip (lightweight-charts crosshair) с временем и значением. |
| Hover на индикаторе health | Tooltip с расшифровкой. |
| Click «Все позиции / Открыть счёт / Подробнее» | Переход на соответствующий экран (text-link, `aria-label`). |
| WS-обновление баланса/позиций | Карточки обновляются без перезагрузки (через WS C1, см. § контракт). |

## 8. Связь с другими экранами

- `/account` — детали баланса, история операций.
- `/trading` (список активных сессий) — открыть/остановить сессию.
- `/sessions/:id` — детали по конкретной сессии.
- `/chart?ticker=X` — клик по позиции.
- `/logs` — клик «Подробнее в логах» из Health.
- `/settings/broker` — пустой state, нет подключения.

## 9. Контракт с backend

> **ARCH-подтверждение (2026-04-25, `arch_design_s7.md` §8):** переиспользуем существующие endpoints; единственная новинка — sparkline для виджета баланса = **новый легковесный endpoint C9**.

- **Баланс (текущее значение):** `GET /api/v1/account/balance` (существует).
- **Sparkline баланса 24h/30d:** `GET /api/v1/account/balance/history?days=30` (контракт **C9 NEW**, BACK1 → FRONT2). Простая агрегация per-day из `daily_stats`.
- **Health:** `GET /api/v1/health` (существует) — потенциально расширяется флагами `tinvest_connected`, `scheduler_running`.
- **Активные позиции:** `GET /api/v1/trading/sessions` (существует, фильтр `status=running`) или `/api/v1/account/positions` — выбираем последний для read-only списка позиций.
- **Realtime обновления:** WS C1 `/ws/trading-sessions/{user_id}` — обновление позиций/PnL без polling (после интеграции FRONT1 в W1).

## 10. Доступность (a11y)

- Каждая карточка — `<article role="article" aria-labelledby="widget-N-title">`.
- Health-индикаторы — не только цвет, но и текст («GREEN», «ONLINE», «RUNNING») + `aria-label="Circuit Breaker — GREEN, норма"`.
- Цветовой контраст для зелёного/красного — проверить >= 4.5:1 на тёмной теме (пройти Mantine theme audit).
- Keyboard: `Tab` фокусирует карточку (visible focus ring 2px), `Enter`/`Space` активирует основной CTA.
- Sparkline для скринридеров: `aria-label="График баланса за 24 часа, рост 1,01%"`.
- Toggler валют — `<SegmentedControl>` со встроенной a11y.

## 11. Тестовые сценарии для QA

(попадает в `e2e_test_plan_s7.md`, секция 7.7)
