# UX-макет: Бейдж фоновых бэктестов в шапке (для задачи 7.17)

> **Статус:** плейсхолдер. Заполняется UX-агентом на W0 по `prompt_UX.md` задача 5.
> Источники макета:
> - `Sprint_7/sprint_design.md` Приложение D
> - `Sprint_7/prompt_UX.md` секция «Задачи W0 / 5»
> - Память `project_s7_background_backtest.md` — спецификация заказчика дословно

## Содержание (заполнить UX-агенту)

1. **Описание** — пользователь может запускать множественные бэктесты «в фоне» с разными параметрами; модалка `BacktestLaunchModal` НЕ закрывается; бейдж в шапке показывает кол-во активных.

2. **Кнопка «Запустить в фоне» в `BacktestLaunchModal`**
   - Расположение — рядом с «Запустить» (две кнопки).
   - Action — POST /api/v1/backtest, получение `job_id`, запись в `backgroundBacktestsStore`.
   - Toast «Бэктест запущен в фоне».
   - **Модалка остаётся открытой** — пользователь меняет параметры и запускает следующий.

3. **Бейдж в шапке справа**
   - Кружок с числом активных бэктестов (например, `3` в красном кружке).
   - При `count === 0` — бейдж скрыт.
   - Hover показывает label «3 бэктеста в фоне».
   - Click открывает dropdown.

4. **Dropdown по клику**
   - Список активных бэктестов:
     - Тикер
     - Стратегия (имя)
     - Прогресс-бар (0..100%)
     - Время старта (relative: «2 мин назад»)
     - Кнопка «Открыть результат» (после `status === 'done'`)
     - Кнопка «×» (отмена — опционально, если backend поддерживает).
   - Источник данных — `backgroundBacktestsStore` + WS подписка `/ws/backtest/{job_id}` для каждого.

5. **States dropdown**
   - Empty: «Нет активных бэктестов» (но dropdown тогда не открывается).
   - Loading: skeleton на каждой строке при subscribe.
   - Error: красная иконка на строке + reason при `status === 'error'`.

6. **Lifecycle**
   - `done` event — запись остаётся в dropdown ещё N секунд (10s?), потом убирается, можно посмотреть результат.
   - `error` — остаётся до явного закрытия пользователем.

7. **ASCII-mockup** шапки с бейджем + dropdown открытым.

8. **Контракт с backend** — `WS /ws/backtest/{job_id}` (контракт C2). Параллельные jobs — отдельные подключения.

9. **Тестовые сценарии для QA** — toast при клике «в фоне»; бейдж инкрементируется; модалка не закрывается; несколько фоновых параллельно — все в dropdown с прогрессом.
