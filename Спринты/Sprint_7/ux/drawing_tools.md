# UX-макет: Инструменты рисования на графике (для задачи 7.6)

> **Статус:** плейсхолдер. Заполняется UX-агентом на W0 по `prompt_UX.md` задача 3.
> Источники макета:
> - `Sprint_7/sprint_design.md` Приложение D
> - `Sprint_7/prompt_UX.md` секция «Задачи W0 / 3»

## Содержание (заполнить UX-агенту)

1. **Описание** — тулбар графика с 3 инструментами для разметки.
2. **Расположение тулбара** — слева вертикально или сверху горизонтально (выбрать).
3. **Инструмент «Трендовая линия»** — два клика (точка1, точка2), preview во время рисования.
4. **Инструмент «Горизонтальный уровень»** — один клик, persistent priceLine.
5. **Инструмент «Прямоугольная зона»** — drag, rect overlay.
6. **Persist** — localStorage по ключу `{user_id, ticker, tf}`.
7. **States** — рисую (курсор-крестик, preview), выделено (selected), редактирую (drag handle), удалить (контекстное меню или Del).
8. **ASCII-mockup** тулбара + примеры состояний.
9. **Контракт с backend** — нет (всё в localStorage).
10. **Тестовые сценарии для QA** — провёл линию → перезагрузил → линия осталась; перенос линии drag&drop; удаление через контекстное меню.

## Stack Gotchas-замечания

- lightweight-charts series cleanup (gotcha из `Develop/stack_gotchas/INDEX.md`) — соблюсти при unmount компонента.
- При смене ticker/tf — выгружать рисунки текущего, загружать рисунки нового.
