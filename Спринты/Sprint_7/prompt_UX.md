# Задачи W0 (приоритет, выполняется ДО старта DEV)

## 1. `Sprint_7/ux/dashboard_widgets.md` (для 7.7)

- **Виджет «Баланс»** — крупное значение, sparkline за день, индикатор ↑/↓ цветом, разница в %.
- **Виджет «Health»** — светофор: CB-state (зелёный/жёлтый/красный), T-Invest connection (online/offline), Scheduler (running/stopped). Подписи рядом.
- **Виджет «Активные позиции»** — мини-графики (или sparkline) для каждого открытого тикера, текущий P&L цветом.
- **Сетка/расположение** — 3 колонки на desktop, 1 колонка на mobile.
- **States** — loading (skeleton), error (текст + retry), empty («Нет активных сессий»).
- ASCII-mockup или ссылка на изображение.

## 2. `Sprint_7/ux/wizard_5steps.md` (для 7.8)

Шаги:

1. **Приветствие** — логотип, текст «Добро пожаловать в MOEX Terminal».
2. **Дисклеймер о рисках** — длинный текст + ОБЯЗАТЕЛЬНАЯ галка «Я прочитал и понимаю». Кнопка `Дальше` disabled пока галка снята.
3. **Выбор брокера / режим** — radio: paper / real, dropdown брокера (T-Invest пока единственный).
4. **Настройка уведомлений** — Telegram (bot_token field), Email (опционально), In-app (всегда вкл).
5. **Финиш** — «Готово, начать работу».

- **Прогресс-бар** сверху (5 точек или линия).
- **Кнопки** Назад / Дальше / Завершить (Mantine Stepper).
- **States** каждого шага: пустой, заполненный, ошибка валидации.

## 3. `Sprint_7/ux/drawing_tools.md` (для 7.6)

- **Тулбар графика** — расположение (вертикально слева от графика или горизонтально сверху).
- **3 инструмента:** трендовая линия, горизонтальный уровень, прямоугольная зона.
- **Persist** — localStorage по ключу `{user_id, ticker, tf}`.
- **States** — рисую (курсор-крестик, preview), выделено (выбрано после клика), редактирую (drag handle), удалить (контекстное меню или Del).

## 4. `Sprint_7/ux/backtest_overview_analytics.md` (для 7.16)

Источник: память `project_s7_interactive_zones.md`.

- **Гистограмма «Распределение P&L»** — бакеты по % (-2%, -1.5%, ..., +4%, +4.5%), красные столбики = убытки, зелёные = прибыли. Пунктирные вертикальные линии: средний убыток и средняя прибыль.
- **Donut «Win/Loss/BE»** — 3 сегмента: Победы (зелёный), Убытки (красный), Безубыточность (жёлтый). В центре donut: число сделок. Справа: легенда с кол-вом и %.
- **Расположение** — два графика рядом на вкладке «Обзор» бэктеста.
- **Источник данных** — `trades[]` из `BacktestResult`.

## 5. `Sprint_7/ux/background_backtest_badge.md` (для 7.17)

Источник: память `project_s7_background_backtest.md`.

- **Бейдж в шапке справа** — кружок с числом активных бэктестов, hover показывает label «N бэктестов в фоне».
- **Dropdown по клику** — список с прогрессом каждого: тикер, стратегия, прогресс-бар, время старта, кнопка «Открыть результат» (после `done`).
- **Toast** «Бэктест запущен в фоне» при клике «в фоне» в `BacktestLaunchModal`.
- **Модалка `BacktestLaunchModal`** — НЕ закрывается после клика «в фоне», параметры можно изменить и запустить ещё.
- **States dropdown** — пустой («Нет активных бэктестов»), загрузка, список.

## 6. `Sprint_7/ux/ai_commands_dropdown.md` (для 7.19)

Источник: `.claude/plans/purring-herding-badger.md`.

- **Dropdown при вводе `/`** в input AI-чата — 5 команд: `/chart`, `/backtest`, `/strategy`, `/session`, `/portfolio`.
- **Подсказки** для каждой команды: краткое описание + пример («`/chart EURUSD 1H` — добавить контекст графика»).
- **Рендер «контекстного блока»** в сообщении пользователя — chip с типом контекста и id (например, «📊 Chart: EURUSD, 1H»). Кликабельный → переход к ресурсу.
- **States** — ввод (dropdown открыт), выбор команды (требуется аргумент), подтверждение (chip отображается перед отправкой), ошибка (id не принадлежит юзеру).
