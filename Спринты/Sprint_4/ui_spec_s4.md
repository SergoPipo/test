## 1. AI-помощник в Strategy Editor

### 1.1 Концепция

AI-чат — **отдельный полноэкранный режим** внутри вкладки «Редактор» в StrategyEditPage. Переключение между Blockly workspace и AI-режимом — **свайп-анимация** (slide left / slide right).

**Вкладка «Код» убирается.** Вкладка «Редактор» отвечает за всё: блочная схема + AI-помощник.

### 1.3 Анимация перехода

```
Blockly → AI:
  Blockly workspace: translateX(0) → translateX(-100%)
  AI-режим:          translateX(100%) → translateX(0)
  Длительность: 300ms, ease-in-out

AI → Blockly (кнопка «Применить к блокам» или «← Вернуться»):
  AI-режим:          translateX(0) → translateX(100%)
  Blockly workspace: translateX(-100%) → translateX(0)
  Длительность: 300ms, ease-in-out
```

### 1.4 Левая панель: описание стратегии (40%)

**Mantine-компоненты:**
- `Textarea` — редактируемое текстовое поле, monospace шрифт
- `Button` variant="filled" — «Применить к блокам»
- `Alert` variant="light" color="yellow" — предупреждения парсера

**Поведение:**
- Описание **редактируемое вручную** (не read-only)
- AI обновляет описание в реальном времени (подсвечивать изменённые секции)
- Пользователь может попросить AI привести текст к формату шаблона
- При нажатии «Применить к блокам»:
  1. Описание парсится Template Parser (S3)
  2. Блоки загружаются в Blockly workspace
  3. Код перегенерируется Code Generator
  4. AI-режим закрывается (анимация → Blockly)
  5. Toast: «Блоки обновлены» (Mantine `notifications.show`)

**Предупреждения:**
```
⚠ Секция "Стоп-лосс" не распознана — нет соответствующего блока
⚠ Индикатор "VWAP" не поддерживается в текущей версии
```

### 1.6 Контекст блоков для AI

При открытии AI-режима автоматически собирается контекст:

```typescript
const aiContext = {
  currentBlocks: workspace.getBlocksJson(),      // текущие блоки
  currentDescription: strategy.description,       // текущее описание
  generatedCode: strategy.generated_code,         // код стратегии
  availableBlocks: [                               // что можно предложить
    { type: "indicator", options: ["SMA","EMA","RSI","MACD","BB","ATR","Stochastic","Volume"] },
    { type: "condition", options: ["Crossover","CrossUnder","GreaterThan","LessThan","InZone"] },
    { type: "signal", options: ["EntryLong","ExitLong"] },
    { type: "filter", options: ["TimeFilter","DayOfWeekFilter"] },
    { type: "money_mgmt", options: ["PositionSizer","MaxPositions","StopLoss","TakeProfit"] },
  ],
};
```

AI **проверяет** при каждом ответе:
- Каждый индикатор → есть блок в системе?
- Каждое условие → реализуемо в Blockly?
- Формат описания → соответствует шаблону?
- Предупреждает, если что-то нереализуемо

AI **не делает**:
- Не меняет блоки напрямую
- Не генерирует произвольный Python-код
- Не добавляет параметры, которых нет в блоках

- `Progress` value={67} size="lg" color="blue" animated
- `Text` size="sm" — текущая дата
- `Text` size="sm" c="dimmed" — elapsed time
- `Button` variant="subtle" color="red" — Отменить

**Поведение:**
- Прогресс обновляется через WebSocket (канал `backtest:{id}`)
- При завершении (`backtest.completed`) — автоматический переход на `/backtests/:id`
- При ошибке (`backtest.failed`) — Alert с текстом ошибки и кнопкой «Закрыть»

**Mantine-компоненты:**

- `Select` — модель (динамический список зависит от типа)
