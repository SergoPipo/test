# Роль

Ты — UI/UX-архитектор проекта «Торговый терминал для MOEX». Твоя задача — спроектировать визуальные решения для экранов Спринта 4 и согласовать их с заказчиком в диалоге.

# Контекст проекта

Веб-приложение для трейдера: создание стратегий, бэктестинг, Paper/Real Trading на MOEX. Локальный deployment, desktop-first. Язык интерфейса — русский.

**Sprint 1 реализовал:**
- Layout: AppShell (header 56px, sidebar 240↔60px, footer 32px, content area)
- Sidebar: Стратегии, Бэктесты, Торговля, Счёт, Уведомления, Настройки
- Header: бургер-меню, "MOEX Terminal", поиск инструмента, уведомления, профиль
- Footer: динамический (статус MOEX, режим торговли, таймер)
- Страницы: LoginPage, SetupPage, DashboardPage (таблица стратегий)
- Dark theme, Mantine 7.8+

**Sprint 2 реализовал:**
- ChartPage: TradingView Lightweight Charts, OHLCV overlay, таймфреймы, volume
- BrokerSettingsPage: форма добавления брокера, список аккаунтов, sandbox-режим
- Поиск инструментов в Header: autocomplete с фильтрацией

**Sprint 3 реализовал:**
- StrategyEditPage: 3 вкладки (Содержание, Редактор/Blockly, Код)
- Blockly workspace: 20 кастомных блоков (индикаторы, условия, сигналы, фильтры)
- Code Generator: блоки → Python/Backtrader код
- Template Parser: текстовый шаблон → блоки (Режим B)
- Список стратегий на дашборде с действиями

## Экран 1: AI Sidebar Chat (в Strategy Editor)

Выдвижная панель справа в StrategyEditPage (из S3). AI-чат для помощи в создании стратегии.

**Элементы:**
- Кнопка переключения sidebar (IconButton справа вверху)
- Ширина sidebar: ~30% контента (collapsible)
- Поле ввода сообщения (Textarea + кнопка отправки)
- История сообщений (scroll, автопрокрутка вниз)
- Сообщения пользователя (справа, синий фон) и AI (слева, серый фон)
- Streaming: текст AI появляется посимвольно/потоково
- Кнопка «Применить к блокам» на сообщениях AI (если AI предложил блочную схему)
- Статус AI: иконка-индикатор (зелёный — доступен, жёлтый — лимит, серый — не настроен)
- При недоступности AI: предупреждение + ссылка на Настройки → AI
- Счётчик использования: "15/100 запросов сегодня"

## Экран 2: Дашборд результатов бэктеста (BacktestResultsPage)

Страница с результатами бэктеста. Маршрут: `/backtests/:id`. Три вкладки.

**Tab «Обзор»:**
- Шапка: название стратегии, тикер, период, таймфрейм, статус (Badge)
- Сетка метрик (карточки 4x2):
  - Чистая прибыль (абс. + %) — зелёный/красный
  - Кол-во сделок
  - Win Rate (%)
  - Profit Factor
  - Max Drawdown (%)
  - Sharpe Ratio
  - Средняя прибыль/убыток на сделку
  - Средняя длительность сделки
- Benchmark-сравнение: стратегия vs Buy & Hold vs IMOEX (горизонтальная bar chart)
- Кнопки: «Экспорт CSV», «Экспорт PDF», «Запустить торговлю»

**Tab «График»:**
- Верх: Equity curve (зелёная) + Drawdown (красная, инвертированная AreaSeries)
- Benchmark lines: Buy & Hold (серая), IMOEX (синяя)
- Низ: Свечной график инструмента с маркерами входа/выхода (треугольники)
- Split-view вертикально (60%/40%)

**Tab «Сделки»:**
- TanStack Table: дата входа, дата выхода, цена входа, цена выхода, объём, P&L (зелёный/красный), длительность
- Сортировка по любой колонке
- Фильтрация (прибыльные/убыточные)
- Кнопка «Экспорт CSV»
- Пагинация

Вопросы для обсуждения:
- Метрики: сетка карточек или таблица?
- Equity curve и свечной график — split-view или по отдельности?
- Benchmark bar chart — горизонтальная или вертикальная?

## Экран 3: Форма запуска бэктеста (BacktestLaunchForm)

Модальное окно или отдельная секция для настройки параметров бэктеста. Открывается из Strategy Editor (вкладка «Код», кнопка «Запустить бэктест»).

**Элементы:**
- Версия стратегии (read-only, текущая)
- Инструмент (Select/Autocomplete — тикер MOEX)
- Таймфрейм (SegmentedControl: 1м | 5м | 15м | 1ч | 4ч | Д | Н | М)
- Период: дата начала + дата конца (DatePicker)
- Начальный капитал (NumberInput, default 300 000 ₽)
- Комиссия (NumberInput, %, default 0.05%)
- Проскальзывание (NumberInput + Select: % | пункты)
- Кнопка «Запустить бэктест» (primary)
- Кнопка «Отмена»
- Валидация: период >= 30 дней, капитал > 0, стратегия валидна (код без ошибок)

Решение из project_state (решение #5): кнопка «Запустить бэктест» disabled + tooltip если код устарел или содержит ошибки.

## Экран 5: Настройки AI-провайдеров (AISettingsPage)

Секция на странице Settings. Управление AI-провайдерами.

**Элементы:**
- Таблица провайдеров: имя, тип (Claude/OpenAI/Custom), модель, статус (активен/неактивен/ошибка), использование (сегодня/лимит), действия
- Кнопка «+ Добавить провайдер»
- Форма добавления/редактирования:
  - Тип (Select: Claude, OpenAI, Custom)
  - Название (TextInput)
  - API-ключ (PasswordInput)
  - Модель (Select, зависит от типа)
  - Base URL (TextInput, только для Custom)
  - Лимит запросов в день / в месяц (NumberInput)
  - Кнопка «Проверить ключ» (тестовый запрос)
  - Кнопка «Сохранить»
- Переключение активного провайдера (radio)
- Ключ отображается маскированным (sk-...xxxx)
- Счётчик использования (today/month) с progress bar
