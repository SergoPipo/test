# 004: Стриминг свечей в реальном времени (текущая формирующаяся свеча)

> **Приоритет:** Высокий
> **Источник:** Замечание при тестировании Sprint 5 — график не показывает текущую формирующуюся свечу
> **Дата:** 2026-04-08

---

## Проблема

T-Invest API (`GetCandles`) и MOEX ISS возвращают только **завершённые** свечи. Текущая формирующаяся свеча не включается в ответ. Из-за этого:

- Дневной ТФ: видна только вчерашняя свеча, а не сегодняшние торги
- Часовой ТФ: последняя видимая свеча — 1–3 часа назад
- Минутный ТФ: отставание на 1–2 минуты

Для торгового терминала это критично — трейдер должен видеть актуальные данные.

## Предлагаемое решение

### Архитектура

```
T-Invest gRPC Stream
    │
    ▼
MarketDataStreamManager (новый)
    │ subscribe_candles(ticker, timeframe)
    │ → получает candle updates в реальном времени
    ▼
Event Bus → WebSocket → market:{ticker}:{tf}
    │
    ▼
CandlestickChart (фронтенд)
    │ wsChannel = "market:SBER:D"
    │ → обновляет последнюю свечу в реальном времени
    ▼
Пользователь видит текущую формирующуюся свечу
```

### Шаги реализации

1. **MarketDataStreamManager** (бэкенд, новый сервис)
   - Управляет подписками на стриминг свечей
   - При первом запросе графика для тикера — открывает gRPC stream
   - Публикует обновления в Event Bus → WebSocket
   - Автоматически закрывает stream при отсутствии подписчиков (таймаут)

2. **Фронтенд: подключение wsChannel**
   - `ChartPage` передаёт `wsChannel={`market:${ticker}:${timeframe}`}` в `CandlestickChart`
   - Компонент через WebSocket получает обновления текущей свечи
   - Обновляет последний элемент в массиве `candles` (или добавляет новый)

3. **Fallback без T-Invest**
   - Если нет брокерского аккаунта — периодический polling MOEX ISS (каждые 30 сек)
   - Менее точно, но лучше чем ничего

## Критерии готовности

- [ ] При открытии графика видна текущая формирующаяся свеча
- [ ] Свеча обновляется в реальном времени (OHLCV меняется при каждом тике)
- [ ] При смене таймфрейма стриминг переподписывается
- [ ] При закрытии графика стриминг отписывается
- [ ] Fallback на polling MOEX ISS если нет T-Invest
- [ ] Учтены выходные торговые сессии T-Invest

## Риски

| Риск | Влияние | Митигация |
|------|---------|-----------|
| Лимиты T-Invest на стриминг | Отключение при превышении | Rate limiter + переподключение с backoff |
| Много одновременных подписок | Нагрузка на gRPC | Ограничить до 5 активных стримов, LRU |
| Разрыв соединения | Потеря данных | Автопереподключение + дозапрос пропущенных свечей |
