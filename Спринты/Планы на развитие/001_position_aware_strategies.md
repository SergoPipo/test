# 001: Позиционные блоки в Blockly-стратегиях

> **Приоритет:** Высокий
> **Источник:** Обсуждение Sprint 5 — поле `max_concurrent_positions` бесполезно без поддержки в стратегиях
> **Дата:** 2026-04-08

---

## Проблема

Текущие Blockly-стратегии **не знают о состоянии позиции**. Генератор кода жёстко вшивает:

```python
# Вход — ТОЛЬКО если позиции нет
if not self.position:
    self.buy(...)

# Выход — ТОЛЬКО если позиция есть
if self.position:
    self.close()
```

Из 20 существующих блоков ни один не даёт доступа к:
- Наличию открытой позиции
- Цене входа
- Текущему размеру позиции
- Нереализованному P&L

### Что невозможно сейчас

- «Если уже в позиции и цена упала на 5% — докупи»
- «Закрой половину позиции при прибыли +3%»
- «Передвинь стоп-лосс в безубыток после +2%»
- «Увеличь позицию, если сигнал повторился»

### Следствие

Поле `max_concurrent_positions` в форме запуска торговой сессии не имеет практического смысла — стратегия всегда работает в режиме «одна позиция, вход/выход».

---

## Предлагаемое решение

### Новые блоки (категория «Позиция»)

| Блок | Тип | Возвращает | Описание |
|------|-----|-----------|----------|
| Has Position | Условие | bool | Есть ли открытая позиция |
| Position Size | Значение | number | Количество лотов в позиции |
| Entry Price | Значение | number | Средняя цена входа |
| Unrealized P&L % | Значение | number | Нереализованная прибыль/убыток в % |
| Unrealized P&L ₽ | Значение | number | Нереализованная прибыль/убыток в рублях |
| Position Direction | Условие | bool | Long или Short |

### Новые сигнальные блоки

| Блок | Описание |
|------|----------|
| ADD TO POSITION | Докупить N лотов к существующей позиции |
| PARTIAL EXIT | Закрыть N% или N лотов из позиции |
| MOVE STOP LOSS | Переместить стоп-лосс на указанный уровень |
| TRAILING STOP | Динамический стоп с отступом N% от максимума |

### Примеры стратегий, которые станут возможны

**Пирамидинг (докупка на просадках):**
```
ЕСЛИ: SMA(20) > SMA(50)                    ← тренд вверх
  И: НЕТ позиции
  → ВХОД (Long, 30% депозита)

ЕСЛИ: Есть позиция
  И: Нереализ. P&L < −3%                   ← просела на 3%
  И: SMA(20) > SMA(50)                     ← тренд всё ещё вверх
  → ДОКУПИТЬ (20% депозита)

ЕСЛИ: Нереализ. P&L > +5%
  → ЧАСТИЧНЫЙ ВЫХОД (50% позиции)
  → ПЕРЕДВИНУТЬ стоп-лосс в безубыток
```

**Трейлинг-стоп:**
```
ЕСЛИ: Crossover(SMA(10), SMA(30)) вверх
  → ВХОД (Long)
  → TRAILING STOP (отступ 2%)
```

### Frontend

| Файл | Изменения |

## Риски и ограничения

| Риск | Влияние | Митигация |
|------|---------|-----------|
| Сложность песочницы | Доступ к `self.position` может открыть вектор атаки | Whitelist-подход: разрешить только конкретные атрибуты |
| FIFO при частичном закрытии | Налоговый расчёт усложняется | Обновить Tax Service для поддержки partial exits |
| Обратная совместимость | Старые стратегии используют хардкод `if not self.position` | Версионирование: v1 (текущий) и v2 (позиционный) генератор |
