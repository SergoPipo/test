# 005: Real-time обновление торговых сессий через WebSocket

> **Приоритет:** Средний
> **Источник:** Тестирование Sprint 6 — карточки торговых сессий обновляются только по polling (10 сек)
> **Дата:** 2026-04-22

---

## Проблема

Страница «Торговля» отображает карточки активных торговых сессий с информацией о позициях, сделках и P&L. Текущая реализация обновляет данные **polling-ом раз в 10 секунд**. Это приводит к:

- Задержка до 10 секунд между реальным событием (открытие/закрытие сделки) и отображением на экране
- Лишняя нагрузка на backend — запросы идут даже когда ничего не меняется
- При большом количестве активных сессий — множество одновременных REST-запросов

---

## Текущее состояние

Backend **уже публикует** все необходимые события через EventBus:
- `order.placed` — ордер выставлен (канал `trades:{session_id}`)
- `trade.filled` — сделка исполнена
- `trade.closed` — сделка закрыта
- `cb.triggered` — Circuit Breaker сработал
- `session.started` / `session.stopped` — старт/остановка сессии

WebSocket-эндпоинт `/ws` уже поддерживает подписку на каналы `trades:{session_id}`.

---

## Предлагаемое решение

### Фронтенд
1. При открытии страницы «Торговля» — подписаться на WS-каналы `trades:{session_id}` для каждой активной сессии
2. При получении события — обновить соответствующую карточку в Zustand store без REST-запроса
3. При добавлении/удалении сессии — подписаться/отписаться от WS-канала
4. Polling убрать или оставить как fallback с увеличенным интервалом (60 сек)

### Данные для обновления по WS-событиям
| Событие | Что обновить в карточке |
|---------|------------------------|
| `trade.filled` | Позиция (direction, lots, entry_price), счётчик сделок |
| `trade.closed` | P&L, win/loss счётчики, позиция → "нет" |
| `cb.triggered` | Показать предупреждение на карточке |
| `order.placed` | Статус: "Ожидание исполнения" |

### Backend
Доработки не требуются — EventBus и WS уже функционируют.

## Что затрагивает

| Компонент | Изменения |

| Backend | Без изменений |

---

## Критерии готовности

- [ ] При открытии сделки — карточка обновляется < 1 секунды (без polling)
- [ ] При закрытии сделки — P&L обновляется мгновенно
- [ ] При Circuit Breaker — на карточке появляется индикатор
- [ ] Polling убран или переведён на 60-секундный fallback
- [ ] Отписка от WS при уходе со страницы (memory leak protection)
