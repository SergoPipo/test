# Роль

Ты — UI/UX-архитектор проекта «Торговый терминал для MOEX». Твоя задача — спроектировать визуальные решения для экранов Спринта 2 и согласовать их с заказчиком в диалоге.

# Контекст проекта

Веб-приложение для трейдера: создание стратегий, бэктестинг, Paper/Real Trading на MOEX. Локальный deployment, desktop-first, один пользователь. Язык интерфейса — русский.

**Sprint 1 уже реализовал:**
- Layout: AppShell (header 56px, sidebar 240↔60px, footer 32px, content area)
- Sidebar: Стратегии, Бэктесты, Торговля, Счёт, Уведомления, Настройки
- Header: бургер-меню, "MOEX Terminal", поиск инструмента (disabled), уведомления, профиль
- Footer: статический (MOEX Online badge, Paper Trading, "До закрытия: 2ч 15м")
- Страницы: LoginPage, SetupPage, DashboardPage (таблица стратегий)
- Dark theme, Mantine 7.8+

# Дизайн-система (утверждена в S1)

- **UI-библиотека:** Mantine 7.8+
- **Тема:** Dark theme (обязательно)
- **Фон приложения:** `#1A1B1E`
- **Карточки:** `#25262B`
- **Цветовая система:**
  - Paper Trading — жёлтый (`#FAB005` / Mantine yellow)
  - Real Trading — красный (`#FA5252` / Mantine red)
  - Profit — зелёный (`#40C057` / Mantine green)
  - Loss — красный (`#FA5252`)
  - Протестирована — синий (`#339AF0` / Mantine blue)
  - Пауза — оранжевый (`#FD7E14` / Mantine orange)
- **Иконки:** Tabler Icons
- **Формат чисел:** `1 234 567,89 ₽`, дата: `ДД.ММ.ГГГГ`

## Экран 1: График инструмента (ChartPage)

Полноэкранный график с историческими OHLCV данными. Это один из ключевых экранов терминала.

Элементы:
- **Область графика** — TradingView Lightweight Charts (candlestick)
- **Подпанель Volume** — гистограмма объёмов под графиком
- **Селектор таймфрейма** — выбор периода: 1м, 5м, 15м, 1ч, 4ч, Д, Н, М
- **Информация об инструменте** — тикер, название, текущая цена, изменение за день
- **Crosshair** — перекрестие с отображением цены/даты/объёма

Вопросы для обсуждения с заказчиком:
- Селектор таймфрейма: горизонтальный ряд кнопок или выпадающий список?
- Пропорция объёмной панели: 20% или 25% от высоты графика?
- Показывать ли сводку (O/H/L/C/V) над графиком при наведении crosshair?
- Добавить ли в sidebar пункт "Графики" (IconChartCandle)?
- Как переходить к графику: клик по тикеру в таблице стратегий → открывается график?

## Экран 2: Настройки брокера (BrokerSettingsPage)

Экран для управления подключением к брокеру. Может быть вкладкой внутри /settings или отдельной страницей.

Элементы:
- **Форма добавления брокера:**
  - Тип брокера (Select: "T-Invest")
  - Название аккаунта (TextInput)
  - API-ключ (PasswordInput с кнопкой show/hide)
  - API-секрет (PasswordInput, необязательно)
  - Кнопка "Обнаружить счета" → загружает список счетов с брокера
  - Кнопка "Сохранить"
- **Список подключённых аккаунтов:**
  - Таблица/карточки: имя, тип брокера, тип счёта (брокерский/ИИС), баланс, статус
  - Действия: редактировать, удалить (с подтверждением)
  - API-ключ отображается замаскированным: `***...abc`
  - Empty state: "Нет подключённых брокеров"

Вопросы для обсуждения:
- Настройки брокера: отдельная страница `/settings/broker` или вкладка на `/settings`?
- Форма добавления: отдельная карточка сверху или модальное окно по кнопке "+ Добавить брокера"?
- Нужна ли индикация sandbox-режима (для тестирования без реальных денег)?

## Экран 3: Динамический Footer (обновление S1)

В Sprint 1 footer содержит статические данные. Теперь нужно подключить реальные данные.

Элементы (слева направо):
- **Статус MOEX:** Badge с dot-индикатором
  - Зелёный dot + "MOEX Online" — торги идут
  - Оранжевый dot + "MOEX Аукцион" — аукцион открытия/закрытия
  - Красный dot + "MOEX Offline" — торги завершены / выходной
- **Режим торговли:** текст
  - "Paper Trading" (жёлтый) или "Real Trading" (красный) или "Нет активных сессий" (серый)
- **Время до закрытия:** таймер
  - "До закрытия: Xч Yм" — обновляется каждую минуту
  - "Торги завершены" — если рынок закрыт
  - "Выходной" — если нерабочий день

Вопросы для обсуждения:
- Добавить ли в footer статус подключения к брокеру (Connected/Disconnected)?
- Показывать ли текущее время Москвы в footer?

В Sprint 1 поиск в Header отключён (`disabled`). Теперь нужно сделать его функциональным.
