# Спринт 1: Фундамент (недели 1-2)

## Цель

Каркас приложения, инфраструктура, аутентификация, UI-kit.

## Критерии завершения

- [ ] Авторизация работает (регистрация, логин, JWT refresh)
- [ ] Дашборд отображается по утверждённому дизайну (dark theme, sidebar, header)
- [ ] CI pipeline проходит (lint + type check + unit tests)
- [ ] Health endpoint `/api/v1/health` возвращает 200
- [ ] Все 23+ SQLAlchemy моделей созданы, первая Alembic-миграция применяется
- [ ] Скрипты install.sh и start.sh работают
- [ ] **Все backend-тесты проходят (`pytest tests/ -v` — 0 failures)**
- [ ] **Все frontend-тесты проходят (`pnpm test` — 0 failures)**

## Подход к тестированию (S1-S2)

На спринтах 1 и 2 **тестирование — ответственность разработчиков**:
- Каждый DEV-агент пишет unit-тесты для своего кода
- ARCH при ревью **запускает все тесты** и проверяет их качество
- Отдельный QA-агент появляется начиная со Спринта 3

| Агент | Что тестирует |
|-------|--------------|
| DEV-1 | Health endpoint, config, exceptions |
| DEV-2 | Модели (создание, типы Numeric, UNIQUE, FK, audit_log triggers), миграции |
| DEV-3 | Форматирование ru-RU, authStore, Sidebar навигация |
| DEV-4 | Ручная верификация скриптов (install.sh, start.sh) |
| DEV-5 | Регистрация, логин, JWT, брутфорс-защита, API endpoints |
| **ARCH** | **Запуск ВСЕХ тестов, проверка coverage, интеграционная проверка** |
