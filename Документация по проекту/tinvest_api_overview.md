# T-Invest API — Общее описание

> Источник: [developer.tbank.ru/invest/api](https://developer.tbank.ru/invest/api), [GitHub: RussianInvestments/investAPI](https://github.com/RussianInvestments/investAPI)
>
> Дата фиксации: 2026-03-26

---

## Что это

T-Invest API — gRPC-интерфейс для взаимодействия с торговой платформой Т-Инвестиции. Предоставляет:

- Размещение торговых ордеров (лимитные, рыночные, стоп-заявки)
- Получение рыночных данных и исторических котировок
- Информацию по портфелю и доходности
- Проверку торговых алгоритмов на исторических данных (sandbox)

## Токены доступа

Токен — набор символов с зашифрованной информацией о владельце и правах доступа. Передаётся в metadata каждого gRPC-запроса.

### 4 типа токенов

| Тип | Описание | Торговля |
|-----|----------|----------|
| **Readonly** | Доступ только на чтение: портфель, расписание, котировки, история | Нет |
| **Full-access** | Полный доступ ко всем методам API | Да |
| **Привязанный к счёту** | Ограничен конкретным счётом, уровень прав настраивается (readonly или full-access) | Зависит от прав |
| **Sandbox** | Только для тестирования в песочнице. Эмуляция торгов на реальных рыночных данных | Только sandbox |

### Получение токена

1. Авторизоваться в настройках Т-Инвестиций
2. Отключить «Подтверждение сделок кодом»
3. Выпустить токен (для биржи **или** для песочницы — выбирается при создании)
4. Скопировать (отображается только один раз)

### Срок действия

- Активен **3 месяца** с момента последнего использования
- Становится недействительным при: смене статуса клиента, завершении сессий, блокировке счёта
- Отозвать можно в личном кабинете

### Ограничение

Сделки свыше 30 млн рублей требуют подтверждения по СМС и недоступны через API.

## Лимиты (Rate Limits)

### Общие ограничения

- Рекомендация: не более **50 запросов в секунду** по всем счетам и токенам суммарно

### Лимиты по сервисам (запросов в минуту)

| Сервис | Лимит |
|--------|-------|
| MarketDataService | 600 |
| InstrumentsService | 200 |
| OperationsService | 200 |
| OrdersService (общий) | 100 |
| PostOrder | 300 |
| PostOrderAsync | 450 |
| CancelOrder | 100 |
| UsersService | 100 |
| StopOrdersService | 50 |
| **SandboxService** | **200** |
| GetHistory (архивы) | 30 |
| Генерация отчётов | 5 |

### Грейды (1-5)

Грейд зависит от объёма ордеров и процента исполнения за последние 30 дней:

| Ресурс | Грейды 1-4 | Грейд 5 |
|--------|------------|---------|
| Stream-подключения (MarketData) | 16 | 64 |
| Stream-подключения (Orders) | 10 (1-3), 16 (4) | 16 |
| Stream-подключения (Operations) | 2 | 2 |

### Stream-ограничения

- Макс. **300 одновременных подписок** (свечи + стаканы + сделки) на одно stream-соединение
- Макс. **100 запросов подписки в минуту** (иначе `SUBSCRIPTION_STATUS_TOO_MANY_REQUESTS`)
- Свечи: не чаще 1 раза в 300мс + свеча закрытия периода
- Стакан: интервал обновления 100мс

## Идентификаторы инструментов

Методы API принимают `instrumentId`, который может быть:
- **instrument_uid** (рекомендуется)
- **FIGI** (устаревший, но поддерживается)
- Комбинация **ticker + class_code**
