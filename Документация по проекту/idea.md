Описание идеи реализуемого приложения.

1. ЧТО ДЕЛАЕТ ПРИЛОЖЕНИЕ?
Веб - приложение, которое предоставляет возможность проаналищировать некую торговую стратегию, описанную словами пользователя, на конкретных финансовых инструментах. Система должна иметь возможность скачать историю торгов фин инструменнтами и сохранить ее в базе данных, чтобы не скачивать аналогичные данные повторно. Также в БД будут хрнаиться сведения о результатах бэктестов по каэждой стратегии с заявленными при бэктестах параметрах. Приложение должно иметь графический интерфейс по аналогии с tradingView для отобрадения того, как применялась заявленная стратегия, где были точки входа и выхода, а также иметь статистику по количеству сделок, профит-фактор и масимальная просадка.
Следующая важная часть приложения - это возомжность работать с брокеров в "боевом" режиме, то есть открывать и закрывать сделки с учетом указанной стратегии. При этом все сделки необзодимо отправлять в телеграм зарегистрированному пользователю. Для этого организовать бота, который будет уведомлять о том, какая сделка открыта, на какую сумму, или какая сделка закрыта и с какой прибылью/убытком. Также должна быть возможность поулчить сведения о всех открытых сделках. Должна быть возможность закрыть все сделаки единовременно
При этом для каждой открытой сделки должна быть возможность открыть текущий график и посмтреть ход торговли в разных временных периодах

2. КТО ПОЛЬЗОВАТЕЛЬ?
Пользователь - трейдер, понимающий процессы торговли финансовыи инструментами, но не профессионально занимающийся такой деятельностью

3. КАК ВЫГЛЯДИТ ТИПИЧНЫЙ СЕАНС РАБОТЫ?
Пользователь открывает страницу и авторизуется в системе. Далее представлена таблица, содержащая список торговых стратегий, который были использованы для анализа, с указанием того, на каких инструментах они были протестированы с результатми. Если данная стратегия запущена в реальной торговле, имеется отметка об этом с указанием брокера, финансового инструмента и текущего показателя профитности. Пользователь может вызывать команду для добавления новой стратегии. Либо перейти в результатм уже имеющеся стратегии. Либо перейти к отображению текущих торгов на базе этой стратегии.
При описании новой стратегии пользователю предосиавляется возможность сгенерировать новое описание, которое далее может быть скорректировано и оптимизированы с точки зрения грамотности ее построения при помощи AI-помощника. Далее пользователю может выполнить бэктест, выбрав инструмент, период и таймфрейм. В случае, если данные об истории торгов уже есть в БД, то система использует их для анализа, если нет, то обращается к брокеру для того, чтобы получить такие исторические данные. После того, как такие данные будут получены в отдельном дэшбрде должен отобразиться результат бэктеста. Результаты бэктестов автоматически сохраняются в БД, чтобы их можно быо в любой момент открыть и выполнить при использовании новых вводных данных.
Также отдельной командой можно подключить такую стратегию к боевому режиму торгов на конкретном броекре и инструменте. При этом нужно создать телеграм бота, который будет отправлять подробную информацию обо всех открываемых сделках, события выхода из сделок, а также статистику применения стратегии.

4. КАКИЕ ДАННЫЕ ХРАНЯТСЯ/ ОТКУДА БЕРУТСЯ?
В БД хранятся отдельно:
    - учетные данные пользователей системы
    - стратегии с их описанием (по каждому пользователю отдельно)
    - исторические данные о графиках финансовых инструментов
    - результаты применения стратегий (бэктесты) за конкретные периоды по конкретным финансовым инструментам (по каждому пользователю отдельно)
    - необходимые сведения для идентификации у брокеров (по каждому пользователю отдельно)
    - сведения об открытых и завершенных сделках у брокеров (в разрезе пользователей)
    - дополнительная необходимая аналитика, которую я не учел в описанных выше пунктах
Сведения о пользователях формируются при регистрации в реализуемой системе. Стратегии вводятся и описываются пользователями в ручном режиме. Исторические данные подгружаются от брокеров и хранятся в том формате, который бужет удобен и оптимален для проведения бэктестов в рамках системы. Результаты выполненных бэктестов по стратегиям сохраняются автоматическом режиме. При подключении некоей стратегии к конкретным инструментам у брокера все сведения об открытых и завершенных сделках сохраняются в БД.

5. Похожие инструменты, которыми я пользовался?
Существует приложение Veles, которое умеет прорабатывать стратегии и выполнять бэктесты и далее подключаться к биржам для ведения автотрейдинга. Формально то, что я хочу создать напоминает это решение, но создается с целью применения данной системы для рынке ЦБ РФ. ДЛя этих целей часто используется Backtrader, и соответстенно нам надо реализовать для его использования графический интерфейс.

6. ЧТО ОБЯЗАТЕЛЬНО ДОЛЖНО БЫТЬ В ПЕРВОЙ ВЕРСИИ?
Нужна возмодность авторизоваться, подключиться к брокеру, чтобы загрузить исторические данные для построения графика. Нужна возомжность указать стратегию и выполнить бэктест на имеющихся исторических данных.
