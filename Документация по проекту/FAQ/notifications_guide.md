## 1. trade_opened — Открытие сделки

**Severity:** `info`
**Триггер:** SessionRuntime публикует событие `order.placed` в EventBus после того, как ордер на вход в позицию принят брокером.
**Связанная сущность:** `session` → `/trading/sessions/{id}`

## 4. cb_triggered — Circuit Breaker

**Severity:** `critical`
**Триггер:** Модуль Circuit Breaker (`app/circuit_breaker/`) обнаруживает нарушение одного из 9 лимитов риска. Торговая сессия автоматически ставится на паузу. Проверки выполняются перед каждым ордером и после каждой сделки.
**Связанная сущность:** `session` → `/trading/sessions/{id}`

**Примеры срабатывания:**
- Дневной убыток превысил лимит (напр. -3% от капитала при лимите 3%)
- Превышено максимальное количество сделок за день
- Размер одной позиции слишком большой
- Drawdown портфеля превысил порог

### Лента (/notifications)

```
🚨  Circuit Breaker сработал
    MA Cross (SBER, 1h): превышен дневной лимит убытков
    Убыток за день: -3.2% (лимит: 3.0%)
    Сессия приостановлена автоматически.
    18.04.2026, 16:15                                  [Перейти к сессии →]
```

### Колокольчик (drawer)

```
🚨  Circuit Breaker сработал
    MA Cross (SBER): дневной убыток -3.2%
    1 мин. назад
```

## 7. session_recovered — Восстановление после рестарта

**Severity:** `info`
**Триггер:** Backend перезапустился (деплой, обновление, crash). При старте функция `restore_all()` находит сессии со статусом "active"/"paused" в БД и восстанавливает их: подписки на котировки, мониторинг сигналов. Это **не** ручное снятие с паузы пользователем.

## 8. connection_lost — Потеря соединения с брокером

**Severity:** `critical`
**Триггер:** StreamMultiplexer теряет gRPC-соединение с T-Invest API (сбой сети, обслуживание на стороне брокера). Backend пытается переподключиться автоматически (exponential backoff). Торговые сессии продолжают работать на backend, но не получают котировки и не могут размещать ордера.
**Связанная сущность:** `broker` → `/settings?tab=broker`

## 13. price_alert — Ценовое оповещение

**Severity:** `success`
**Триггер:** Пользователь устанавливает ценовой алерт на графике (кнопка "Ценовое оповещение" → тикер + условие "выше"/"ниже" + целевая цена). Backend мониторит каждую свечу из MarketDataStream. Когда цена закрытия пересекает уровень алерта — срабатывает уведомление, алерт деактивируется.
**Связанная сущность:** `instrument` → `/chart?ticker={ticker}`

### Лента (/notifications)

```
✅  Ценовое оповещение
    SBER достиг 270.00 ₽ (условие: выше 270.00)
    Текущая цена: 270.45 ₽
    18.04.2026, 15:42                                  [Открыть график →]
```

### Колокольчик (drawer)

